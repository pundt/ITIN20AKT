UI_Reiseboerse_Graf.Models Namensraum
enthält alle Klassen, um die einzelnen Websites für die Oberfläche zu erzeugen
Klassen
  KlasseBeschreibung
BuchungAnzahlModel
Model, das die Daten enthält um die passenden Felder für die Dateneingabe der Reisenden zu erzeugen Es wird für jeden Erwachsenen und jedes Kind eine Eingabemaske erzeugt
BuchungAnzeigenModel
Model zur Anzeige von Buchungen in der Buchungshistory für den Benutzer
BuchungenModel
Model zur Erfassung der Daten der Reisenden (Eingabemaske)
BuchungErwachsenModel
Erbt von BuchungenModel, validiert das Geburtsdatum für Erwachsene
BuchungGesamtModel
Viewmodel, dass alle Informationen eines Buchungsprozesses beinhaltet
BuchungHinzufuegenModel
Viewmodel, das alle Informationen zur Erfassung der Reisedaten beinhaltet# Erzeugung der korrekten Anzahl an Eingabemasken
BuchungKindModel
Erbt von BuchungenModel, validiert das Geburtsdatum für Kinder
BuchungVerwaltenModel
Model zur Ansicht der Buchung für den Mitarbeiter
FilterModel
Das Model dient zur Filterung der Reisen
KundenAnlegenModel
Das Model zur Registrierung eines Benutzers/Kunden
KundenModel
KundenModel, erbt von KundenAnlegenModel
LandModel
Das Model, das den Namen eines Landes und die eindeutige ID beinhaltet
LoginModel
Das Model, das an den Login gebunden ist und validiert ob Passwort und Benutzername richtig sind
OrtModel
Das Model, das den Namen eines Ortes und eine eindeutige ID beinhaltet
ReiseAnlegenModel
legt das Model für neue Reisen an
ReiseAnzeigenModel
Viewmodel für die Detailansicht einer Reise von dort aus kann ein Buchungsprozess gestartet werden
ReisedatumModel
es werden alle Daten (Datum) erfasst und deren Anzahl / bei mehreren Datenangaben kann man wieder auf die Seite zurückkommen
ReisedetailModel
ReisedetailModel beinhaltet alle Detailinformationen einer Reise erbt von ReiseModel
ReisedurchfuehrenModel
Hier wird das Datum erhoben für die Datumtabelle und ReiseAnzahl für ReiseAnzahltabelle
ReiseLadenModel
Viewmodel für die Startseite mit Anzeige aller Reisen
ReiseModel
Das Model, das alle grundsätzlichen Informationen zu einer Reise beinhaltet
ReiseVerwaltenModel
Das Model, das in Verwaltung der Reisen verwendet wird. Es dient zur Auflistung aller Reisen
StornoModel
Das Model, das in der Verwaltung für die Mitarbeiter verwendet wird
TextsucheModel
Model zur Textsuche auf der Oberfläche
UnterkunftdetailModel
Model für Unterkunftsanzeige/Erstellung
VerpflegungModel
Model für Verpflegung (beinhaltet ID und Bezeichnung) wird für befüllen der Dropdownlisten verwendet
ZahlungModel
Model zur Eingabe einer Zahlung (Formular)
ZahlungsartModel
Model für die Zahlungsart beinhaltet ID und Bezeichnung wird für Befüllen der Dropdownliste verwendet
